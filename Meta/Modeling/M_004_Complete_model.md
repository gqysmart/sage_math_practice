# M-000 · 完整建模框架（Complete Modeling Framework）

> **定位说明**
> 本文是对 `M-000_Modeling_Layers.md` 的结构性补全与统一版本。
> 目标不是增加“更多术语”，而是给出一套**可长期使用、可自检、可迁移**的建模总体框架。

---

## 0. 核心立场（Core Position）

**建模不是表达世界，而是为不同理解主体构造“可操作的结构”。**
因此，建模必然是：

* 多层的（layered）
* 有选择的（selective）
* 带约束的（constrained）
* 面向目标的（goal-oriented）

不存在“无假定、无裁剪、无视角”的模型。

---

## 1. 建模的纵向递进链路（Modeling Layers）

这是你在原始 M-000 中确立的**主线结构**：

> **问题 → 数学模型 → 计算模型 → 算法 → 语言实现**

关键判断：

* 这不是“同一事物的不同表述”
* 而是**连续、不可跳过的再建模过程**
* 每一层都面向一个不同的**理解主体**

| 层级   | 主要理解主体    |
| ---- | --------- |
| 问题   | 人 / 领域专家  |
| 数学模型 | 逻辑系统      |
| 计算模型 | 抽象计算机     |
| 算法   | 程序模型      |
| 语言实现 | 编译器 + 运行时 |

---

## 2. 横向关键元素总览（Structural Elements）

仅靠“对象、属性、关系”不足以描述模型的真实结构。
一个**完整模型**至少由三大类关键元素共同构成。

```
模型结构 = 本体元素
        + 运作元素
        + 裁剪/选择元素
```

---

## 3. A 类：本体元素（Ontology · What Exists）

> 回答：**模型世界中“有什么”**

### 3.1 对象（Object / Entity）

* 被视为整体的基本单元
* 实体、变量、节点、实例

### 3.2 属性（Attribute / Feature）

* 对象的可观测或潜在描述量
* 标量 / 向量 / 结构化

### 3.3 状态（State）

* 对象在某一时刻的属性配置
* 状态空间中的一个点

### 3.4 关系（Relation）

* 对象之间的结构性连接
* 函数、约束、图结构、依赖关系

> 本体元素定义的是模型的**静态存在论**。

---

## 4. B 类：运作元素（Mechanics · How It Works）

> 回答：**模型如何运行、演化、推理**

这是大量“概念模型”失败的原因所在。

### 4.1 规则 / 定律（Rule / Law）

* 状态转移规则
* 决策规则
* 更新函数

```
s_{t+1} = f(s_t, a_t)
```

### 4.2 过程 / 动力学（Process / Dynamics）

* 连续 / 离散
* 时间驱动 / 事件驱动
* 马尔可夫 / 非马尔可夫

### 4.3 因果结构（Causality）

* 影响方向
* 可干预性（do-operator）
* 决策与控制的前提

### 4.4 推理 / 计算机制（Inference / Computation）

* 搜索、优化、估计、采样
* 前向 / 反向 / 近似推理

> 没有 B 类元素，模型只能“描述”，不能“行动”。

---

## 5. C 类：裁剪与选择元素（Selection · What We Ignore & Why）

> 回答：**为什么模型是这样，而不是那样**

这一类不是“存在物”，而是**元结构（meta-structure）**。

### 5.1 建模目标（Objective / Question）

* 模型试图回答什么？
* 评估标准是什么？

### 5.2 范围与边界（Scope / Boundary）

* 哪些变量进入模型
* 哪些被排除在外

### 5.3 表示选择（Representation）

* 向量 / 图 / 张量 / 符号
* 连续 / 离散
* 精确 / 近似

### 5.4 约束（Constraint / Feasible Set）

* 合法状态的边界
* 物理 / 逻辑 / 资源约束

### 5.5 等价关系与不变量（Equivalence / Invariant）

* 哪些差异被视为“同一”
* 商空间、同构、不变性

### 5.6 简化假定（Assumptions）

* 为可操作性而主动忽略的因素
* 线性性、独立性、稳定性等

### 5.7 适用域与失效条件（Validity / Failure Mode）

* 模型在什么条件下成立
* 何时必须停止使用

> **模型的本质，是一组系统性的忽略规则。**

---

## 6. 横向 × 纵向的统一视角

每一层建模，都是在**同一组关键元素上重新取值**，以适配新的理解主体。

| 关键元素 | 数学模型 | 计算模型 | 算法   | 实现            |
| ---- | ---- | ---- | ---- | ------------- |
| 对象   | 变量   | 数据结构 | 元素   | 内存对象          |
| 状态   | 状态空间 | 内存快照 | 中间状态 | Runtime state |
| 规则   | 方程   | 更新逻辑 | 步骤   | 控制流           |
| 等价   | 同构   | 表示等价 | 剪枝   | Cache key     |
| 假定   | 连续   | 离散   | 近似   | 容错            |

---

## 7. 方法论自检清单（Modeling Checklist）

在任何项目中，可以反问：

1. 我现在是在为谁建模？
2. 本体、运作、裁剪三类元素是否齐全？
3. 哪些假定是隐含的？能否被显式写出？
4. 模型的失效条件是什么？
5. 是否在低层过早引入了高层假设？

---

## 8. 总结性一句话

> **一个模型不是“世界的缩影”，而是一个在目标约束下、
> 为特定理解主体定制的、可运行的认知装置。**
