一、数学问题也有“状态空间”

只是这个状态不是：

节点

数组下标

而是：

命题 / 表达式 / 结构

例如：

证明：
n(n+1)/2 是前 n 项和

状态空间：

S = {P(1), P(2), P(3), ...}


状态：

P(n)：命题在 n 成立


转移：

P(n) → P(n+1)


这就是：

数学归纳法 = 状态转移

二、例 1：数学归纳法（最典型的“状态转移”）

证明：

1 + 2 + ... + n = n(n+1)/2


状态：

S_n = 命题在 n 成立


转移：

S_n → S_{n+1}


方式：

S_{n+1} = S_n + (n+1)


这和 DP 完全同构：

dp[n+1] = dp[n] + (n+1)


区别：

DP：算值

数学：证明恒成立

三、例 2：欧几里得算法（GCD）

数学问题：

gcd(a, b)


状态：

(a, b)


转移：

(a, b) → (b, a mod b)


直到：

b = 0


这其实是：

状态空间：整数对

转移：模运算

终止条件：b = 0

完全是：

状态搜索


只是：

正确性由数学定理保证

不需要尝试不同路径

四、例 3：高斯消元（线性代数）

问题：

解 Ax = b

状态：

矩阵形态


转移：

行变换


过程：

原矩阵
→ 上三角
→ 单位阵


这其实是：

状态：矩阵

转移：初等行变换

目标：标准形

只是：

每一步“唯一”

不需要搜索

五、例 4：概率递推

经典：

期望步数问题

状态：

E[x]：在 x 位置的期望


转移：

E[x] = 1 + pE[x+1] + qE[x-1]


这就是：

状态空间：x

转移：概率转移

不变量：期望方程

其实：

DP + 数学证明

六、数学推理与 DP 的本质区别
	DP	数学
是否计算值	是	不一定
是否需要遍历	可能	不需要
是否存在转移	有	有
转移来源	子问题	定理/性质
正确性来源	枚举 + 递推	证明
七、真正关键的一点

数学问题不是没有“状态转移”，而是：

转移是“必然的”，不是“选择的”

对比：

搜索
有很多路
要选

DP
有递推
要算

数学
只有一条逻辑链
必须这么推

八、统一视角（最核心）

三者其实是同一个模型的不同层次：

搜索 = 多路径状态转移
DP = 单路径值传播
数学 = 必然逻辑转移

九、一个非常深的统一公式

所有问题都可以写成：

state_{t+1} = F(state_t)


区别：

类型	F 是什么
搜索	策略
DP	递推式
数学	定理
